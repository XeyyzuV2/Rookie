## Code Quality and Stability

These commits improve the project's tooling chain, standardise code style, and remove dead code to reduce the maintenance surface.

### chore: add configuration files for oxfmt and oxlint

Replaced the legacy ESLint setup with [oxlint](https://oxc.rs) and [oxfmt](https://oxc.rs/docs/guide/usage.html#oxfmt), a Rust-based linter and formatter that runs significantly faster. The new configuration enables rules from `typescript`, `unicorn`, `oxc`, `react`, `nextjs`, `jsx-a11y`, and `import` plugin categories and treats all correctness issues as errors.

```json
{
  "plugins": [
    "typescript",
    "unicorn",
    "oxc",
    "react",
    "nextjs",
    "jsx-a11y",
    "import"
  ],
  "categories": {
    "correctness": "error"
  }
}
```

The corresponding npm scripts were added so you can lint and format with a single command:

```json
{
  "scripts": {
    "lint": "oxlint .",
    "lint:fix": "oxlint . --fix",
    "format": "oxfmt --write .",
    "format:check": "oxfmt --check ."
  }
}
```

### chore: update documentation layout and metadata formatting

Standardised quote usage across the codebase to double quotes and tightened up OpenGraph/Twitter metadata blocks for better social-sharing previews. Keywords and author fields were reformatted for consistency, and several component files were cleaned up as part of the pass.

### chore: remove outdated server configuration details

Deleted the server configuration section from the docs, including environment-variable examples and SSL/TLS configuration that no longer reflected the project's deployment setup. Keeping the documentation aligned with the actual stack prevents confusion for new contributors.

### chore: remove unused browser and server files

Removed `.source/browser.ts` and `.source/server.ts` -- generated files that were no longer referenced by the build pipeline. These were artifacts of an earlier iteration of the fumadocs source setup and were safe to delete.

## Documentation Infrastructure

These commits lay the foundation for the documentation site, including the fumadocs source loader, search API, and the initial page structure.

### feat: init base project

Scaffolded the Next.js project with TypeScript, Tailwind CSS v4, and fumadocs. Established the base directory layout under `content/docs/` and wired up the MDX loader via `source.config.ts`:

```ts
import { defineDocs, defineConfig } from 'fumadocs-mdx/config';

export const docs = defineDocs({
  dir: 'content/docs',
});

export default defineConfig();
```

The source loader maps icon names to Lucide and React Icons components for the documentation sidebar:

```ts
export const source = loader({
  baseUrl: "/docs",
  source: docs.toFumadocsSource(),
  icon(icon) {
    if (!icon) return;
    if (icon in icons) {
      return createElement(icons[icon as keyof typeof icons]);
    }
  },
});
```

## User Interface and Experience

Improvements to theme handling, navigation, scroll behaviour, and visual polish.

### feat: update navbar theme toggle functionality

Replaced the ad-hoc `mounted` state check with a proper `useHydrated` hook built on React's `useSyncExternalStore`. This eliminates the flash of incorrect theme on first render and ensures the toggle icon matches the actual resolved theme (light or dark) rather than the stored preference.

```tsx
function useHydrated() {
  return useSyncExternalStore(
    () => () => {},
    () => true,
    () => false
  );
}

const activeTheme = hydrated ? resolvedTheme : undefined;
const nextTheme = activeTheme === "dark" ? "light" : "dark";
const ThemeIcon = activeTheme === "dark" ? Sun : Moon;
```

The navbar scroll detection was also moved from a passive listener to an active one by calling `handleScroll()` on mount, preventing a stale initial state.

### feat: update smooth scrolling

Integrated Lenis for smooth, hardware-accelerated scrolling across the documentation site. The smooth-scroll component wraps the main content area and provides a consistent scrolling experience that respects the user's `prefers-reduced-motion` setting.

### feat: adding favicon; adding logo on docs sidebar title

## Bug Fixes

### fix: improve focus handling in InputGroupAddon

Refactored the `focusInput` callback in the `InputGroupAddon` component to use a typed `querySelector` call instead of an untyped chain. The previous one-liner relied on implicit typing and could silently fail; the new version explicitly casts to `HTMLInputElement | HTMLTextAreaElement` and safely optional-chains the `.focus()` call.

```tsx
const focusInput = (element: HTMLDivElement) => {
  const input =
    element.parentElement?.querySelector<
      HTMLInputElement | HTMLTextAreaElement
    >("input, textarea");

  input?.focus();
};
```
